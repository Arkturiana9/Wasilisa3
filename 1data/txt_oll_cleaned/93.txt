
Введение

В современных условиях развития промышленности повышение эффективности и качества производства становится одним из ключевых приоритетов для промышленных предприятий. Для достижения этой цели активно применяются информационные системы (ИС), обеспечивающие автоматизацию процессов и контроль за качеством выпускаемой продукции.
Целью выпускной квалификационной работы является создание ИС для отдела контроля качества.
Для выполнения этой цели были поставлены следующие задачи:
	разработка технического задания на программный продукт;
	проведение анализа предметной области;
	проектирование структуры базы данных ИС и архитектуры приложения;
	определение целей и задач проекта;
	описание деятельности в рамках проекта, определение этапов и сроков выполнения;
	определение ресурсных потребностей проекта;
	определение стоимости проекта;
	разработка интерфейса и проведение внутреннего тестирования ИС;
	формирование внутренней документации по результатам выполнения работ.

	Теоретические аспекты разработки информационной системы

	1.1 Классификация информационных технологий и систем

Информационная система для отдела контроля качества промышленного предприятия представляет собой совокупность программно-аппаратных средств, баз данных и персонала, предназначенную для сбора, хранения, обработки и предоставления информации, необходимой для управления качеством продукции на всех этапах производственного процесса

Основные функции такой информационной системы:
	автоматизация процессов контроля качества сырья, материалов и готовой продукции;
	ведение баз данных по поставщикам, клиентам, рекламациям и претензиям;
	анализ данных о качестве и формирование отчетности;
	поддержка принятия решений по улучшению качества.
Классификация информационных систем для контроля качества.
Системы управления качеством (СУК) - предназначены для автоматизации процессов контроля качества продукции на всех этапах производства.
     СУК позволяют вести учет данных о качестве, проводить анализ и принимать решения по улучшению качества.
Лабораторно-информационные системы (ЛИМС) - предназначены для автоматизации работы испытательных лабораторий.
ЛИМС позволяют ускорить процесс аналитического контроля, снизить количество ошибок, обеспечить идентификацию образцов и протоколирование всех этапов работы с ними.
Системы управления производством (MES) - предназначены для оперативного управления и контроля производственными процессами, включая контроль качества
MES позволяют отслеживать параметры качества в режиме реального времени и оперативно реагировать на отклонения.
Системы бизнес-аналитики (BI) - предназначены для анализа информации о предприятии и его окружении
BI позволяют понять движущие силы предприятия, предсказать влияние решений на качество и принимать обоснованные управленческие решения.
Таким образом, информационная система для отдела контроля качества промышленного предприятия должна включать в себя элементы СУК, ЛИМС, MES и BI для комплексной автоматизации процессов управления качеством

Для того, чтобы использовать весь потенциал и возможности информационной системы, ее необходимо адаптировать под решаемые задачи и под особенности тех условий, в которых она эксплуатируется. Для этого предусмотрены определенные операции и процедуры по настройке и отладке ИС. Таким образом достигается наибольшая эффективность и оптимальность ИС.
Перед любой информационной системой стоит определенный ряд задач, которые она должна решать. Эти задачи отличаются в зависимости от назначения ИС, а также от специфики организации или предприятия, где она эксплуатируется. В общем случае, ИС обеспечивают решение следующих наиболее типовых и распространенных задач:
	непрерывный мониторинг системы в целом, а также отдельных режимов ее функционирования;
	формирование прогнозов и возможных вариантов развития событий с составлением алгоритмов действия;
	непрерывный контроль степени актуальности используемых информационных ресурсов, в том числе информационных баз;
	подстройка и отладка объекта автоматизации в соответствии с действующим условиями работы.
В зависимости от функциональных особенностей, а также от поставленных задач, информационные системы могут быть объединены в определенные группы.

1.2База данных SQL

 (Structured Query Language) - это структурированная коллекция данных, организованная в виде таблиц, где каждая таблица содержит строки и столбцы, а доступ к данным осуществляется с помощью SQL запросов. SQL является языком запросов, который используется для управления данными в реляционных базах данных, таких как Microsoft SQL Server.
Использование базы данных SQL для разработки информационной системы для отдела контроля качества промышленного предприятия позволяет:
Хранить и организовывать данные о качестве продукции, контроле качества, рекламациях и других аспектах контроля качества[1].
Обеспечивать быстрый доступ к информации, анализировать данные и генерировать отчеты.
Реализовывать функционал системы с помощью SQL запросов и пользовательских функций.
Разрабатывать интерфейс информационной системы для удобного взаимодействия с данными и выполнения необходимых операций.
Таким образом, база данных SQL является основой для хранения и управления данными в информационной системе отдела контроля качества промышленного предприятия, обеспечивая эффективное управление информацией, анализ данных и принятие обоснованных решений по улучшению качества продукции.




1.3 Управление качеством в промышленном производстве

Управление качеством в промышленном производстве играет ключевую роль в обеспечении высокого уровня качества продукции и удовлетворения потребностей клиентов. Для выпускной квалификационной работы по теме разработки информационной системы для отдела контроля качества промышленного предприятия важно рассмотреть следующие аспекты управления качеством:
Роль отдела контроля качества: Отдел контроля качества является ключевым звеном в системе управления качеством на промышленном предприятии. Его основной задачей является обеспечение соответствия продукции установленным стандартам качества.
Основные задачи и функции отдела контроля качества: Важно изучить основные задачи, которые выполняет отдел контроля качества, такие как контроль качества сырья и материалов, контроль производственных процессов, анализ и исправление несоответствий, а также мониторинг качества готовой продукции.
Нормативно-правовая база управления качеством: Для эффективного управления качеством необходимо изучить нормативные документы, регулирующие процессы контроля качества на предприятии. Это включает стандарты качества, технические регламенты, законы о защите прав потребителей и другие нормативные акты.
Изучение и углубленный анализ этих аспектов управления качеством в промышленном производстве позволят разработать информационную систему для отдела контроля качества, которая будет эффективно поддерживать процессы управления качеством и обеспечивать высокий уровень качества продукции на предприятии.

2.6 Требования к разработке

При разработке визуального интерфейса информационной системы для отдела контроля качества промышленного предприятия необходимо учитывать ряд ключевых требований, которые обеспечат эффективное функционирование системы и удовлетворят потребности пользователей.
Прежде всего, интерфейс должен быть интуитивно понятным и простым в использовании, чтобы даже пользователи без специального обучения могли легко освоить его функционал. Это подразумевает простую и логичную структуру меню и элементов управления, а также понятное оформление и расположение информационных блоков.
Для обеспечения оперативного контроля за качеством продукции, интерфейс должен предоставлять возможность быстрого доступа к основным функциям и данным системы. Например, кнопки быстрого доступа к ключевым функциям, возможность настройки персональных рабочих пространств пользователей и инструменты для быстрого поиска и фильтрации информации.
Также важно обеспечить адаптивность интерфейса под различные устройства и разрешения экранов. Это позволит пользователям работать с системой на различных устройствах, включая компьютеры, планшеты и мобильные телефоны, что особенно актуально для отдела контроля качества, где часто требуется мобильность и оперативность.
С точки зрения функциональности, интерфейс должен предоставлять широкий спектр возможностей для анализа и визуализации данных о качестве продукции. Это могут быть графики, диаграммы, таблицы и другие элементы, позволяющие наглядно отображать результаты контроля качества и выявлять тенденции и аномалии в производственных процессах.
Важным требованием является также обеспечение безопасности и защиты данных, особенно учитывая чувствительность информации о качестве продукции и производственных процессах. Это включает в себя механизмы аутентификации пользователей, контроль доступа к данным на различных уровнях и шифрование конфиденциальной информации.
Наконец, интерфейс должен быть гибким и легко настраиваемым с целью адаптации под специфические потребности и требования отдела контроля качества. Это может включать в себя возможность настройки отображаемой информации, создание пользовательских отчетов и аналитических инструментов, а также интеграцию с другими системами и сервисами предприятия.?

5 Охрана труда и окружающей среды


5.1 Охрана труда

В настоящее время промышленность нуждается в обеспечении безопасности производственных процессов. С началом XXI века промышленная безопасность вышла на новый этап развития. Аналогичная ситуация отмечается в странах с развитой и развивающейся экономикой. Акцент на промышленной безопасности поставлен как на ключевом условии того, что индустриальное предприятие функционирует надежно, экономически эффективно.
Характеризуя промышленную безопасность, нужно понимать, что в таком состоянии общество не пострадает от угрозы жизненно важным интересам, обеспечена надёжная защита социуму и коллективу предприятия, так как нельзя отрицать потенциальной вероятности, что на производственном объекте возникнет аварийная ситуация. Кроме того, промышленно-безопасный объект подразумевает, что все наступившие из-за аварии последствия будут минимальными. Формируя промышленную безопасность, предприятие стремится не допустить аварию, а также, если наступят последствия, добиться быстрого снижения ущерба до минимума.
Итак, на производственных сотрудников могут влиять следующие неблагоприятные производственные факторы:
Психофизиологические факторы.
Санитарно-гигиенические, определяющие внешнюю производственную среду.
Помещение, в котором размещается компьютерная техника, должно соответствовать требованиям безопасности и пожаробезопасности для обеспечения безопасных условий труда.
Планировка и размещение
Размещение должно быть достаточного размера для размещения необходимого оборудования и свободного перемещения персонала.
Расстояние между необходимыми местами должно быть не менее 1,5 метров.
Правильная поза:
Сидеть прямо, спина должна быть ровной и опираться на спинку кресла.
Ноги должны быть слегка расставлены, а стопы полностью опираться на пол.
Колени должны быть немного выше уровня бедра.
Локти должны быть расположены на уровне стола, а предплечья - параллельно поверхности.
Верхняя часть экрана должна располагаться на уровне глаз, расстояние до экрана - около вытянутой руки, но не менее 50 см.
Клавиатура должна быть установлена на уровне локтей с небольшим наклоном.
Излучение экрана
Современные мониторы безопаснее старых моделей, так как имеют меньшие циклы измерения.
Тем не менее, длительная работа за компьютером все равно может отрицательно влиять на зрение, вызывая усталость глаз, проявления и другие симптомы «компьютерного синдрома».
Чтобы свести к минимуму вред от экрана, рекомендуется регулярно делать перерывы, ограничивать время работы за компьютером и использовать специальные очки для чтения.
Электробезопасность
Электрооборудование должно быть исправно и соответствовать требованиям электробезопасности.
Розетки и выключатели должны быть расположены в доступных местах и обеспечивать надежное заземление.
Пожарная безопасность
Помещение должно быть оборудовано стойкой пожарной сигнализацией и пожаротушения.
Должны быть установлены огнетушители, пожарные краны и другие средства пожаротушения.
Пути эвакуации должны быть свободны от препятствий, а обозначения соответствуют знакам.
Освещение и вентиляция
Помещение должно быть оборудовано естественной вентиляцией и искусственной вентиляцией для поддержания необходимого микроклимата.
Освещение должно соответствовать нормам и обеспечивать достаточную видимость для безопасной работы.
Уровень освещенности
Рабочее место должно быть достаточно светлым.
Норма освещенности для офисных помещений - 300-500 лк (люкс).
За работу с документами и за компьютером - не менее 400 лк.
Источники света
Предпочтительно использовать рассеянный свет, исходящий от нескольких источников.
Лампы должны располагаться сбоку от пользователя, чтобы свет падал на внешнюю поверхность под углом 45°.
Лучше использовать лампы с цветовой температурой 3500-4500 К (Кельвин) и индексом цветопередачи (Ra) не менее 80.
Защита от бликов
Экран монитора должен располагаться на возвышении над окнами и источниками света, чтобы избежать попадания бликов.
Окна рекомендуется оборудовать жалюзи или шторами для регулирования естественного освещения.
Лампы должны быть оснащены матовыми плафонами или рассеивателями для предотвращения ослепляющего воздействия.
Охрана труда
В помещении должны быть размещены инструкции по охране труда и технике безопасности.
Персонал должен быть обучен правилам безопасной работы и использованию индивидуальных средств защиты.
Соблюдение требований по утилизации компьютерной техники и обеспечению безопасности помещений позволяет минимизировать риски для здоровья персонала и окружающей среды, а также обеспечить безопасные условия труда.?
Заключение

В рамках выпускной квалификационной работы была разработана информационная система (ИС) для отдела контроля качества промышленного предприятия. Проект включал анализ существующих решений, проектирование базы данных, разработку функциональных модулей для оперативного учета топлива, мониторинга продаж и взаимодействия с клиентами.
Реализованная система позволила автоматизировать процессы сбора, обработки и хранения информации, улучшить точность данных, сократить время на обработку операций и повысить качество обслуживания клиентов. Внедрение ИС способствовало повышению эффективности работы сотрудников и оптимизации управленческих решений на основе актуализированных данных.
